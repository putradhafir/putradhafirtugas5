01. Pengenalan Sealed Class

✦ Pengertian Sealed Class

  • Sealed class adalah jenis class khusus dalam bahasa pemrograman Dart yang membatasi pewarisan (inheritance) hanya dalam satu file yang sama.

✦ Contoh Implementasi

// Nama  : PUTRA DHAFIR NASHRULLAH
// NIM   : 251410009
// Kelas : SI2A

// SEALED CLASS - Membatasi pewarisan hanya dalam satu file
sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

class Loading extends ApiResponse {} // ✅ Ditambahkan

void handleResponse(ApiResponse response) {
  if (response is Success) {
    print("✅ Success: ${response.message}");
  } else if (response is Error) {
    print("❌ Error: ${response.errorMessage}");
  } else if (response is Loading) {
    print("⏳ Loading..."); // ✅ Ditambahkan
  }
}

void main() {
  handleResponse(Success("Data berhasil diambil"));
  handleResponse(Error("Terjadi kesalahan pada server"));
  handleResponse(Loading()); // ✅ Ditambahkan
}

  • Method Overriding adalah fitur dalam Pemrograman Berorientasi Objek (OOP) yang memungkinkan sebuah kelas turunan (subclass) untuk menulis ulang atau mengganti implementasi metode yang sudah ada pada kelas induknya (superclass). Sederhananya, meskipun nama dan parameternya sama, isi instruksi di dalamnya diubah agar lebih spesifik sesuai kebutuhan kelas anak.

6.Pewarisan dan Method Overriding Pada Class Repository (BONUS 1)

✦ Pengertian Class Repository

  • Pewarisan (inheritance) adalah konsep dalam OOP yang memungkinkan class turunan mewarisi atribut dan method dari class induk. Konsep ini membantu mengurangi penulisan kode yang berulang (duplikasi). Class turunan dapat melakukan method overriding, yaitu mendefinisikan ulang method dari class induk sesuai kebutuhan. Pada program, class Repository sebagai class induk memiliki method create, read, update, dan delete. Class UserRepository sebagai turunan mewarisi method tersebut dan melakukan overriding pada method read() untuk menampilkan data user. 
// Nama  : PUTRA DHAFIR NASHRULLAH
// NIM   : 251410009
// Kelas : SI2A

// ABSTRACT CLASS BENTUK GEOMETRI (cukup ditulis 1 kali)
abstract class Bentuk {
  double hitungLuas();
}

class Persegi extends Bentuk {
  double sisi;
  Persegi(this.sisi);

  @override
  double hitungLuas() {
    return sisi * sisi;
  }
}

class Lingkaran extends Bentuk {
  double jariJari;
  Lingkaran(this.jariJari);

  @override
  double hitungLuas() {
    return 3.14159 * jariJari * jariJari;
  }
}

class Segitiga extends Bentuk {
  double alas;
  double tinggi;
  Segitiga(this.alas, this.tinggi);

  @override
  double hitungLuas() {
    return 0.5 * alas * tinggi;
  }
}

void main() {
  Persegi persegi = Persegi(5);
  print("Luas Persegi: ${persegi.hitungLuas()}");

  Lingkaran lingkaran = Lingkaran(7);
  print("Luas Lingkaran: ${lingkaran.hitungLuas().toStringAsFixed(2)}");

  Segitiga segitiga = Segitiga(6, 8);
  print("Luas Segitiga: ${segitiga.hitungLuas()}");
}

8.Penerapan oop Abstraksi

✦ Pengertian Penerapan oop Abstraksi

  • Pengertian Penerapan oop Abstraksi Penerapan OOP: Abstraksi pada Dart merupakan penggunaan salah satu konsep Object-Oriented Programming (OOP) untuk menyederhanakan program dengan menyembunyikan detail implementasi dan hanya menampilkan fungsi utama. Abstraksi membantu programmer dalam membuat program yang lebih terstruktur, mudah dipahami, dan mudah dikembangkan. Dalam bahasa Dart, abstraksi dapat diterapkan menggunakan abstract class, yaitu class yang tidak dapat dibuat objek secara langsung. Abstract class berisi method abstrak yang harus diimplementasikan oleh class turunan sesuai dengan kebutuhan masing-masing. Pada contoh program, abstraksi diterapkan pada class Burung yang memiliki method suara(), kemudian diimplementasikan oleh class BurungBiasa dan Bebek dengan hasil suara yang berbeda.

10.Class Mixin State Pada Dart

✦ Pengertian Class Mixin State Pada Dart

  • Pengertian Class Mixin State Pada Dart Dalam pemrograman Dart, Mixin adalah metode untuk menggunakan kembali (reuse) kode dari satu kelas di beberapa hierarki kelas yang berbeda tanpa harus menjadi kelas induk (pewarisan/inheritance). Mixin memungkinkan pengembang untuk menambahkan fungsionalitas spesifik ke suatu kelas tanpa kerumitan pewarisan tunggal.

mixin Logger {
  String logLevel = "INFO";

  void log(String message) {
    print("[$logLevel] $message");
  }

  void setDebugMode() {
    logLevel = "DEBUG";
  }

  void setErrorMode() {
    logLevel = "ERROR";
  }

mixin Counter {
  int _count = 0;

  int get count => _count;

  void increment() {
    _count++;
    print("Count: $_count");
  }

  void reset() {
    _count = 0;
    print("Count direset ke 0");
  }
}

  void finish() {
    log("Aplikasi selesai");
    log("Total proses: $count");
  }

11.Implementasi Sealed Class dengan Pattern Matching

✦ Pengertian Implementasi Sealed Class

  • Pengertian Implementasi Sealed Class Implementasi sealed class adalah teknik pemrograman untuk membatasi hierarki pewarisan kelas secara ketat, di mana semua subkelas harus didefinisikan dalam berkas yang sama. Ini menciptakan tipe data terstruktur yang membatasi nilai, sering digunakan untuk state management (misal: Loading, Success, Error) yang aman dan rapi. 
// Nama  : Putra Dhafir Nashrullah
// NIM   : 251410009
// Kelas : SI2A

// Sealed class
sealed class Status {
  const Status();
}

class Loading extends Status {
  const Loading();
}

class Loaded extends Status {
  final String data;
  const Loaded(this.data);
}

class Failed extends Status {
  final String error;
  const Failed(this.error);
}

// Function handle status
void handleStatus(Status status) {
  switch (status) {
    case Loading():
      print("Loading...");
    case Loaded(:final data):
      print("Data: $data");
    case Failed(:final error):
      print("Error: $error");
  }
}

void main() {
  handleStatus(const Loading());
  handleStatus(const Loaded("Data berhasil"));
  handleStatus(const Failed("Terjadi error"));
}

12. Studi Kasus State Management

✦ Pengertian Studi Kasus State Management

  • Pengertian Studi Kasus State Management Studi kasus state management adalah analisis mendalam dan terperinci mengenai penerapan metode, strategi, atau pustaka (library) tertentu untuk mengelola data yang berubah-ubah (state) di dalam sebuah aplikasi. Dalam pengembangan perangkat lunak (terutama front-end seperti Flutter, React, atau Next.js), studi kasus ini bertujuan untuk mendemonstrasikan bagaimana data dikelola, diperbarui, dan dibagikan antarkomponen secara efektif dalam skenario dunia nyata 
// Nama  : Putra Dhafir Nashrullah
// NIM   : 251410009
// Kelas : SI2A

// Sealed class
sealed class AppState {}

// State Loading
class LoadingState extends AppState {}

// State Success
class SuccessState extends AppState {
  final List items;
  SuccessState(this.items);
}

// State Error
class ErrorState extends AppState {
  final String message;
  ErrorState(this.message);
}

// Handler (Pattern Matching)
void updateUI(AppState state) {
  switch (state) {
    case LoadingState():
      showLoading();
    case SuccessState(:final items):
      showItems(items);
    case ErrorState(:final message):
      showError(message);
  }
}

// UI Functions
void showLoading() {
  print("⏳ Loading...");
}

void showItems(List items) {
  print("✅ Success! Items:");
  for (int i = 0; i < items.length; i++) {
    print("${i + 1}. ${items[i]}");
  }
}

void showError(String message) {
  print("❌ Error: $message");
}

// Main Program
void main() {
  updateUI(LoadingState());
  updateUI(SuccessState(["Item A", "Item B", "Item C"]));
  updateUI(ErrorState("Gagal memuat data"));
}

📌 KESIMPULAN

  • Sealed Class - Membatasi pewarisan hanya dalam satu file, memungkinkan pattern matching exhaustive.
  • Final Class - Mencegah pewarisan, menjaga konsistensi struktur class.
  • Abstract Class - Kerangka dasar untuk class turunan, tidak bisa diinstansiasi langsung.
  • Mixin - Menambahkan perilaku ke class tanpa pewarisan langsung, bisa memiliki state.
  • Polimorfisme - Method yang sama dapat memiliki perilaku berbeda di class yang berbeda.
  • Inheritance - Pewarisan atribut dan method dari parent ke child class.
  • Abstraksi - Menyembunyikan detail implementasi, hanya menampilkan fungsionalitas utama.

Dengan memahami class modifiers ini, program menjadi lebih terstruktur, aman, dan mudah dikembangkan.
